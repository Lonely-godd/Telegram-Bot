Нужно создать C# desktop-приложение для удобной визуальной настройки Telegram-бота, которое потом генерирует JSON-конфиг для этого бота.

Цель приложения:
Сделать красивый, удобный и понятный редактор настроек бота. Пользователь через интерфейс настраивает структуру информационного Telegram-бота, а приложение автоматически собирает из этого JSON-файл.

Основная логика:
- В боте есть разделы с информацией.
- У каждого раздела могут быть подразделы.
- Все должно поддерживать 2 языка: русский и украинский.
- В начале работы пользователь выбирает язык, который будет использоваться в боте.
- Нужно, чтобы приложение само легко понимало:
  - сколько разделов информации создано;
  - есть ли у разделов подразделы;
  - какие тексты относятся к какому языку;
  - как это все должно быть собрано в JSON.
- Итогом работы приложения должен быть готовый JSON-файл, который можно использовать как основу для Telegram-бота.

Что важно в интерфейсе:
- Все максимально удобно и красиво.
- Нужен понятный визуальный редактор структуры.
- Должны быть кнопки для добавления, удаления, редактирования и перемещения разделов и подразделов.
- Должна быть простая работа с двумя языками в одном интерфейсе.
- Пользователь не должен вручную писать JSON.
- JSON должен собираться автоматически из настроек в приложении.
- Желательно предусмотреть предпросмотр структуры перед сохранением.

Что нужно сделать:
1. Спроектировать удобную структуру данных для разделов, подразделов и языков.
2. Сделать C# приложение с красивым UI.
3. Реализовать редактирование этой структуры через интерфейс.
4. Реализовать генерацию JSON из настроек.
5. Сделать сохранение JSON в файл.
6. Построить структуру так, чтобы ее было легко расширять потом.

Требования к результату:
- Код должен быть аккуратным и рабочим.
- Архитектура должна быть понятной и несложной.
- Структура JSON должна быть продуманной заранее, чтобы бот потом легко читал ее.
- Делай решение так, как будто это должен поддерживать другой разработчик после тебя.

Перед тем как писать код, сначала продумай:
- модель данных;
- структуру JSON;
- структуру UI;
- как пользователь будет добавлять разделы, подразделы и тексты на двух языках.

После этого сгенерируй полный рабочий код приложения.
